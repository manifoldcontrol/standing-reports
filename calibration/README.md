# calibration: the data behind the retention thresholds

Logical closeness and step-label entropy participate in the retention
verdict, so their thresholds are part of the effective verification
policy. This directory publishes the data those thresholds rest on,
sensitivity around the chosen cutoffs, an empirical evaluation of both
metrics, and every quarantined kernel accept.

`python3 recompute.py` re-derives the threshold and sensitivity tables
from the published observations and fails on any mismatch.

## naming

The entropy metric is stored in run records as `dependency_drift` for
schema stability. Its exact name is step-label entropy: the Shannon
entropy, in bits, of the proof's step-label distribution. Logical
closeness is the fraction of trace nodes lying on a path into the
concluding node.

## files

- `calibration_observations.jsonl` -- one row per kernel accept:
  run, theorem, band, DAG source, node count, logical closeness,
  step-label entropy, dangling count. Instrument rule: memoised-walk-era
  records only; earlier tree-walk counts are not comparable and are
  excluded. The simulator baseline was generated by the current
  instrument; the simulator is not a grader (the false band vacuously
  accepts there), and rows carry `band` so consumers can slice.
- `threshold_table.json` -- per-source distribution summaries next to
  the active thresholds.
- `sensitivity.json` -- false-quarantine counts on the accepted corpus
  at caps set to p95, p99, and the observed maximum, next to the active
  cap. "False quarantine" means a kernel-accepted proof the cap would
  demote; the kernel found nothing wrong with it.
- `malformation_eval.json` -- detector evaluation on deliberately
  malformed traces (seeded, class sizes stated in the file).
- `quarantines.json` -- every kernel accept demoted by the gate, with
  the recorded reason, plus revoked retained records.

## headline numbers

Real accepts (n = 112): step-label entropy median 2.27, p95 2.89,
p99 3.12, max 3.68; active cap 3.2. At the active cap exactly one
kernel accept is quarantined (chain_coop_floor, entropy 3.6839, run14):
rate 0.0089. At a p95 cap the rate would be 0.0446 (5 proofs); at the
observed maximum, zero.

Logical closeness on real accepts is degenerate: min = median = 1.0.
Every closed term reaches the sink, so the closeness threshold binds
nothing on the real path; it is a simulator-path check. Structural
integrity on the real path is carried by the dangling-node and
metavariable conjuncts.

Malformation evaluation (600 malformed + 200 clean synthetic traces,
seed in file): edge-drop caught 200/200 (closeness and dangling fire),
uniform label shuffle 167/200 (entropy cap), label collapse 0/200.
Clean false quarantines 0/200. Pooled recall 0.61, precision 1.00.
The label-collapse result is a measured blind spot: the entropy rule
is an upper cap only, and a proof whose steps collapse to one label
passes it. A lower cut is future work and would need its own
calibration.

Scope limit: full proof-term DAGs are not persisted (records keep
counts), so malformation runs on simulator-shaped and synthetic traces.
Perturbing real proof-term DAGs requires persisting them first.
